# OpenVPN Manager — Скрипт управления

> Полнофункциональный скрипт управления OpenVPN сервером — развёртывание одной командой, управление пользователями, ограничение скорости, двухфакторная аутентификация, IPv4/IPv6 двойной стек. Поддерживает все основные дистрибутивы Linux.

## Возможности

| Функция | Описание |
|---------|----------|
| 🛡️ **Двухфакторная аутентификация** | Клиентский сертификат (1-й этап) + логин/пароль (2-й этап) |
| ⚡ **4 серверных процесса** | TUN-UDP / TUN-TCP / TAP-UDP / TAP-TCP |
| 📁 **4 конфигурации клиента** | Автоматическая генерация 4 файлов `.ovpn` для каждого пользователя |
| 🚦 **Индивидуальное ограничение скорости** | Загрузка через shaper, отдача через tc+ifb |
| 🔢 **Контроль одновременных подключений** | Ограничение количества устройств на аккаунт |
| 🌍 **IPv4/IPv6/Двойной стек** | Поддержка всех режимов |
| 🔄 **Режимы маршрутизации** | `split` (только LAN) или `global` (весь трафик) |
| 📋 **Подробное логирование** | Аутентификация, подключения, сессии, назначение IP |
| 🔁 **Автозапуск** | 4 systemd сервиса |
| 🗑️ **Полное удаление** | 8-шаговая полная очистка |
| 🐧 **Мультидистрибутив** | Автоопределение ОС и пакетного менеджера |
| 🔑 **Сертификаты на 99 лет** | Срок действия по умолчанию 99 лет |

## Быстрый старт

```bash
# 1. Скачать
wget https://github.com/StephenJose-Dai/OpenVPNOneKey/archive/refs/tags/V26.6.281056.zip

# 2. Распакуйце
unzip V26.6.281056.zip
```

## Клиентские файлы конфигурации

Для каждого пользователя генерируется 4 файла `.ovpn` в `/www/OpenVPN/clients/<имя>/`:

| Файл | Устройство | Протокол | Рекомендуемое использование |
|------|-----------|---------|---------------------------|
| `*-tun-udp.ovpn` | tun | UDP | **Рекомендуется** — лучшая производительность |
| `*-tun-tcp.ovpn` | tun | TCP | Когда UDP заблокирован файрволом |
| `*-tap-udp.ovpn` | tap | UDP | Мост уровня 2 |
| `*-tap-tcp.ovpn` | tap | TCP | TAP + обход файрвола |

## Архитектура сервера

```
VPN Сервер
├── openvpn-tun-udp.service  порт 1194/UDP  подсеть 10.8.0.0/24
├── openvpn-tun-tcp.service  порт 1194/TCP  подсеть 10.8.1.0/24
├── openvpn-tap-udp.service  порт 1195/UDP  подсеть 10.9.0.0/24
└── openvpn-tap-tcp.service  порт 1195/TCP  подсеть 10.9.1.0/24
```

Каждый процесс использует отдельную подсеть `/24` для предотвращения ARP-конфликтов.

## Аутентификация

```
Клиент подключается
    │
    ▼
【Этап 1】TLS + проверка клиентского сертификата
    ├── Недействительный сертификат  →  Отказ
    ├── Отозванный сертификат        →  Отказ
    └── Действительный ───────────────────────────────────────►
                                                              │
                                                              ▼
                                               【Этап 2】Логин + Пароль
                                                   ├── Пользователь не найден →  Отказ
                                                   ├── Аккаунт отключён      →  Отказ
                                                   ├── Неверный пароль       →  Отказ
                                                   └── OK ──────────────────►  Подключено
```

Пароли хранятся в виде хэша **SHA-256 + случайная соль**. Открытый текст не сохраняется.

## Режимы маршрутизации

### split (по умолчанию)
Только LAN-трафик идёт через VPN. Интернет-трафик использует локальное соединение.

### global
Весь трафик (включая интернет) идёт через VPN.

Изменить режим: `sudo ./openvpn-manager.sh user set-mode alice global`

## Рекомендации по безопасности

1. **Регулярно меняйте пароли** — `user passwd <имя>`
2. **Удаляйте неиспользуемые аккаунты** — `user delete <имя>` также отзывает сертификат
3. **Следите за логами аутентификации** — `log auth` выявляет атаки перебором
4. **Делайте резервные копии PKI** — потеря `easy-rsa/pki/` означает потерю всех сертификатов
5. **Обновляйте пакеты** — `apt upgrade openvpn`
